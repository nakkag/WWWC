#
# フィルタ記述解説
#
# ■書式
#
# 条件	URL	開始文字列	[終了文字列]
#
# の4つをTab区切りで指定します。
#
# ■説明
#
# [条件]
#	以下の文字の組み合わせを指定します。順番と大文字小文字は変わっても構い
#	ません。
#	  例) DI	http://hoge/	<hoge>	</hoge>
#
#	D - 開始文字列から終了文字列の範囲にマッチする部分をHTMLから除去したサ
#	    イズでチェックを行います。
#	S - 開始文字列から終了文字列の範囲にマッチする部分をHTMLから抽出してそ
#	    のサイズでチェックを行います。
#	C - 大文字と小文字を区別しない。
#	R - このフィルタを繰り返し適用する。
#	I - このフィルタが適用されない場合にエラーとして扱わない。
#	E - 抽出でエラーになった場合は本文を空にする。特定文字列の出現をチェック
#	    することができます。
#
#	※ D と S は同時に指定した場合は後に指定した方が有効になります。
#	※ R は D を指定している場合のみ有効です。
#	※ E は S を指定している場合のみ有効です。
#	※ D か S が指定されていない場合は D を指定したことになります。
#
# [URL]
#	フィルタ対象のURLを指定します。
#	* を使用することでサーバやサイトに共通するように指定できます。
#	例) http://*.hoge/*
#
# [開始文字列] [終了文字列]
#	検索する文字列です。* を使用することができます。
#	改行やタブを使う場合は \n \t \\ が使えます。
#	HTML中の \r\n や \r は \n として扱います。
#	検索はタグ単位で行うので文字列内で '<' でタグを開くと '>' でタグを閉じ
#	てください。
# 	<AAA><BBB><CCC> を検索する場合は <AAA*CCC> ではマッチしません。
# 	<AAA*>*<*CCC> のように指定してください。
#	条件に C が指定されていない場合は大文字と小文字を区別して検索します。
#	終了文字列を省略した場合は開始文字列にマッチする部分のみが対象になります
#
#	フィルタを複数指定して最低一つの開始文字列か終了文字列が見つからない場
#	合はエラーとし更新日時に "Filter error" と表示されます。
#	(条件に E か I が指定されているものを除く)
#
# ■その他
#
# WWWCで[Shift]キーを押しながら「ソース表示」を選択するとフィルタを適用した
# ソースを表示します。そのときソース表示画面のタイトルは、
# 	ソース表示(フィルタ) - [URL]
# となります。フィルタの適用に失敗した場合は(フィルタ)の文字が付加されません。
#
# フィルタはアイテムのリクエスト方法が GET の場合のみ有効になります。
# 同じサイトに複数指定すると上から順に実行していき最終的なサイズでチェックを行
# います。
# S を指定して新着部分のみをチェックする対象にしたり、毎回変化のある部分を
# D で除去してチェックを行うなどの使い方ができます。
#
# フィルタは filter.txt という名前で WWWC.DLL と同じフォルダに置いてください。
# WWWC起動中にWWWCにfilter.txtを再読み込みさせたい場合は、WWWC.DLL内のツールの
#	フィルタ再読み込み(&F)
# をWWWCのオプション画面のツールのタブで登録してください。
# (プロトコル指定と同じファイル名で指定しないとうまく動作しないです)
#
# フィルタは1行の終わりに必ず改行を入れてください。
# 最後の行のフィルタで改行が無い場合は、そのフィルタは無視されます。
#
